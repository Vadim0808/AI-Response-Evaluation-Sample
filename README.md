# AI Response Evaluation Sample

## Project Overview

This is a practice portfolio project demonstrating how I evaluate AI-generated responses for helpfulness, accuracy, clarity, safety, and instruction following.

The goal of this project is to compare two AI-generated answers to the same user prompt, identify which response is stronger, explain the reasoning, and provide concise written feedback.

## Target Role

This project is relevant for roles such as:

- AI Response Evaluator
- AI Data Annotator
- AI Trainer
- LLM Evaluator
- Search Quality Rater
- Data Quality Specialist

## User Prompt

How can a small e-commerce store improve product listings and reduce product data errors?

## Response A

A small e-commerce store can improve product listings by creating a consistent structure for every product page. Each listing should include a clear product title, accurate category, complete description, price, SKU, size, color, material, and high-quality images.

To reduce product data errors, the store should use a checklist before publishing each listing. This checklist can include spelling review, duplicate SKU check, missing-field check, category verification, and price format review.

It is also helpful to keep product information in a spreadsheet or catalog management system. This makes it easier to track updates, compare products, and find mistakes before they appear on the website.

## Response B

The best way to improve product listings is to write very exciting descriptions and use as many keywords as possible. Product data errors are not a big problem because customers usually do not notice small mistakes. A store should focus mostly on advertising instead of spending time checking product information.

## Evaluation

### Better Response: Response A

Response A is stronger because it directly answers the user's question and gives practical steps for improving product listings and reducing product data errors. It mentions important listing elements such as product title, category, description, price, SKU, size, color, material, and images.

Response A also explains how to reduce errors by using a checklist, checking for duplicate SKUs, reviewing missing fields, verifying categories, and keeping product data organized in a spreadsheet or catalog system. This advice is specific, realistic, and useful for a small e-commerce store.

Response B is weaker because it focuses mostly on advertising and keyword use, while ignoring the user's request to reduce product data errors. It also incorrectly suggests that product data errors are not a serious issue. In reality, inaccurate product information can create customer confusion, returns, poor user experience, and lower trust.

## Ratings

### Response A: 5/5

Response A is helpful, clear, practical, and follows the user's request well.

### Response B: 2/5

Response B is incomplete and gives poor advice about product data quality. It does not fully answer the user's question.
